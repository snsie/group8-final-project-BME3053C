# Contributing

Thanks for your interest in contributing to this project! To make collaboration smooth, please follow these guidelines.

Getting started
- Use GitHub Codespaces or open this repository in a devcontainer (see `.devcontainer/devcontainer.json`).
- The container includes Lua and `luacheck` for linting.

Branching & workflow
- Work on feature branches named like `feat/<short-desc>` or `fix/<short-desc>`.
- Open a pull request against `main` and use the PR template.

Commit messages
- Use clear, imperative messages: `Add X`, `Fix Y`, `Update Z`.

Running checks locally
- After opening the repo in the devcontainer, run:
```
luacheck .
```

Reporting issues
- Use the issue templates when opening bug reports or feature requests.
